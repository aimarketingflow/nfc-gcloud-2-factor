üìä Component Architecture

üîê NFC Credential Vault

  • File: gcp_nfc_vault.json
  • Encryption: PBKDF2-SHA256 (100k iterations)
  • Key Source: NFC UID (invisible scanning)
  • Content: Google Cloud service account JSON
  • Security: Useless without physical NFC tag

üíª SecureGCPClient

  • File: secure_gcp_client.py
  • Function: NFC-protected Google Cloud API calls
  • Device Binding: Hardware fingerprinting
  • Protection: Blocks all access without NFC

üöÄ AIMF Auth Server

  • File: aimf_auth_server.py
  • Function: NFC fingerprint validation
  • JWT Tokens: 8-hour expiry on pre-authorized devices
  • Claims: nfc_verified, device_id, IP binding
  • Security: Rate limiting, audit logging

‚òÅÔ∏è Google Cloud IAM

  • Policy: Conditional IAM binding
  • Requirement: AIMF JWT with nfc_verified=true
  • Service Account: Restricted permissions
  • Validation: JWT issuer and claims
  • Enforcement: Denies access without valid JWT

üîÑ Data Flow Process

Legitimate User Authentication Flow:
1. User: Initiate Google Cloud API call
2. SecureGCPClient: Request NFC verification
3. User: Scan NFC tag (invisible - no UID display)
4. Client: Derive key from NFC UID + decrypt vault
5. Client: Send NFC fingerprint to AIMF Auth Server
6. AIMF: Validate NFC + pre-authorized device + generate JWT (8hr)
7. Client: Include JWT + service account in Google API call
8. Google: Validate IAM policy (JWT claims + nfc_verified)
9. Google: Execute API operation ‚úÖ

Attacker with Stolen Credentials:

5-Minute Credential Assembly Explanation:
After NFC authentication, decrypted Google Cloud credentials are held in memory for 5 minutes. This allows multiple API calls without re-scanning NFC. After 5 minutes, credentials are automatically cleared from memory for security, requiring a fresh NFC scan to decrypt the vault again. This prevents long-term exposure of assembled credentials while maintaining usability.
